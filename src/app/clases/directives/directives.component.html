Directives (Directivas)

Son las herramientas que nos el framework para poder tener comportamientos adicionales sobre nuestros elementos en el DOM (Document Object Model)


Tipos de directivas:
  • Components Directives: Son las directivas mas comunes y que generalmente se declaran en la etiqueta
  • Structural Directives (Directivas Estructurales): Que sirven para definir que se renderiza en nuestro DOM y por ende puede hacer que nuestro HTML tenga mas o menos codigo y se trabaja con las etiquetas nativas o components directives
  • Attributes Directives (Directivas de atributos): Sirven para aÃ±adir atributos a nuestras etiquetas ya sean nativas de html o nuestros components directives (los que usan los selectores)

Components Directives (Directivas de componentes)

Son aquellas etiquetas que usamos a partir de un selector (por los componentes) o sirven para invocar librerias/modulos


Algunas directivas especiales de angular son:

  • ng-container: Sirve para poner una etiqueta a la cual le podamos aplicar directivas estructurales
  • ng-template: Sirve para poder hacer referencia a que pongamos dicho contenido
  • ng-content: Sirve para indicarle a angular que el contenido que se use en la directiva en padre debe ir dentro de esa etiqueta

Structural Directives (Directivas Estructurales)

Las directivas estructurales son aquellas que nos permiten definir que vamos a renderizar a partir de ciertas expresiones.

*ngIf (v16 e inferiores )/ @ if (v17 y superiores)

Sirven para indicar si un elemento de nuestro HTML se va a renderizar o no, en las versiones anteriores a la 17 se usaba *ngIf apoyado de la libreria commonModule pero a partir de la 17 se usa @ if (unido) y funciona de una manera mas eficiente porque forma parte de manera nativa de angular, adicionalmente tiene funcionalidades aÃ±adidas de manera mas simple. Su sintaxis dependiendo de su version es:


  • 16 y menores: se usa *ngIf="booleano"

  • 17 y mayores: se usa con {{'@if(booleano){...}@else if(booleano){...}@else{...}'}}

    *ngFor (16 e inferiores) / @ for (17 y superiores)

    Es una directiva que sirve para renderizar sobre una iteracion de un array, es decir que cada elemento de una propiedad tendra un template asignado de forma que no tenemos que repetir nuestras etiquetas constantemente por cada elemento. Una vez mas en angular 16 se usaba el commmonModule pero en el 17 se integro como parte de angular haciendolo mas eficiente. Su sintaxis dependiendo de la version es :


    • v16 e inferiores: usamos el *ngFor="let elemento of array; let i = index"
    • v17 y superiores: usamos el @ for(elemento of array; track indice; let i = $index)

    Nota: lo que va despues de los ; son opcionales exceptuando el track en el 17

    ngSwitch (v16 e inferiores) / @ switch (v17 y superiores)

    Permite para definir de un conjunto de opciones donde se hara una comparacion que sea estrictamente igual que template queremos renderizar, una vez mas funciona con commonModule en versiones inferiores a la 16 y forma parte de angular apartir de la 17 gracias al @ switch. Dependiendo de la version su sintaxis es:


    • v16 e inferiores: su sintaxis es:
      • [ngSwitch]="expresion" -> Corresponde a lo que se va a evaluar
      • *ngSwitchCase="valor" -> Corresponde contra que se va a comparar
      • *ngSwitchDefault -> En caso de no corresponder a ningun case se genera este
    • v17 y superiores: su sintaxis es:
      • @ switch(expresion) -> Corresponde a lo que se va evaluar y dentro de las llaves van sus otros elementos
      • @ case(expresionAComparar) -> Corresponde contra que se va a comparar
      • @ default -> Corresponde al valor en caso de no coincidir con ningun case

    Attribute Directives (Directivas de atributos)

    Son aquellas directivas que sirven para poderle dar atributos dinamicos a nuestros elementos y adicionalmente podemos crear directivas de atributos personalizadas.

    ngClass

    El ngClass sirve para permitir modificar todo lo que incumbe a las clases de un elemento, donde podemos agregar, eliminar, modificar directamente su atributo


    Su sintaxis es: [ngClass]="'nombre-clase' : booleano" siendo el valor un objeto constituido de propiedaeds(nombre-clases) y valores boooleanos

    ngStyle


    Es una directiva comunmente usada para hacer modificaciones sobre el estilo de un elemento, de hecho le aplica un style inline


    La sintaxis es: [ngStyle]="'propiedad-css':expresion"
